# gspsiurl2gdrive

### 🛠️ 実現するための全体フロー

1. 外部APIからスプレッドシートへデータ（画像のURL含む）が書き込まれる。
2. スプレッドシートの「変更」を検知して、GAS（プログラム）が自動起動。
3. GASが最新行の画像URLを取得。
4. GASが画像データをダウンロードし、指定した**Google Driveのフォルダ**にPDFとして保存。
5. 保存が完了したら、スプレッドシートに「保存済」とステータスを書き込む（二重処理を防止するため）。

### 💡 確実かつスマートに処理する設計
プログラム側で「画像が入る可能性のある列（例：C列、E列、G列など）」をあらかじめ複数指定しておきます。
ユーザーがフォームを送信した際、プログラムがそれらの列を1つずつチェックします。
「値が入っている、かつ、まだGoogle Driveに保存していないURL」だけを自動で判別してダウンロードします。（空欄の列は自動的にスキップされます）。
保存が成功したら、スプレッドシート上の元のURLを「Google DriveのファイルURL」に直接書き換えます。
この「元のURLをDriveのURLに書き換える」という処理を行うことで、「どの画像が保存済みか」がひと目で分かり、二重処理も完璧に防ぐことができます。

### 📝 具体的な設定手順

#### ステップ 1：Google Driveの準備

1. Google Drive内に、PDFを保存するための専用フォルダを新規作成します。
2. 作成したフォルダを開き、ブラウザのURLバーから**フォルダID**（`https://drive.google.com/drive/folders/〇〇〇` の `〇〇〇` の部分）をコピーして控えておきます。

#### ステップ 2：スプレッドシートにGASコードを貼り付ける

1. 外部APIと連携しているスプレッドシートを開きます。
2. 上部メニューの **「拡張機能」 ＞ 「Apps Script」** をクリックします。
3. もともと入っているコードをすべて消去し、gspreadsheeturl2gdrive.gs の内容を貼り付けます。

4. 貼り付けたら、エディタ上部の「プロジェクトを保存（フロッピーディスクのアイコン）」をクリックして保存します。
5. コード上部の設定エリアにある `IMAGE_URL_COLUMNS` に、画像URLが書き込まれる列番号をすべて指定してください。

* A列＝1、B列＝2、C列＝3、D列＝4、E列＝5 ... と数えます。
* 例：`const IMAGE_URL_COLUMNS = [3, 5, 7];` （C列、E列、G列に画像が入る可能性がある場合）

#### ステップ 3：トリガー（自動実行設定）を追加する【重要】

前述の通り、外部APIからのAPI書き込みは手動編集ではないため、通常の `onEdit` トリガーでは起動しません。以下の手順で「変更時」のインストーラブルトリガーを設置する必要があります。

1. Apps Script画面の左側メニューにある **「トリガー（目覚まし時計のアイコン）」** をクリックします。
2. 右下の **「＋ トリガーを追加」** ボタンをクリックします。
3. 以下のように設定します：
* **実行する関数を選択:** `saveImageToDrive`
* **実行するデプロイを選択:** `Head`
* **イベントのソースを選択:** `スプレッドシートから`
* **イベントの種類を選択:** **`変更時`** （※「編集時」ではなく、必ず「変更時」にしてください）


4. 右下の「保存」をクリックします。
5. Googleアカウントへのアクセス承認を求めるポップアップが表示されるので、画面の指示に従って「詳細」＞「〇〇（安全ではないページ）に移動」＞「許可」の順に進めて承認を完了させます。

#### ステップ4：PC側のファイルが作成されたら印刷するための設定をする
1. SumatoraPDF(https://www.sumatrapdfreader.org/free-pdf-reader)のportable版をダウンロードしautoprintpdf_watcher.ps1と同じフォルダに置く
2. install.bat を右クリックして「管理者として実行」 すれば自動設定がすべて完了

### 💡 この設計（コード）の安全対策ポイント

1. **`onChange`（変更時）トリガーの採用**
外部システムからデータが追記された際に、確実にプログラムを起動させるための必須設定です。
2. **`Utilities.sleep(2000)` による同期ズレ防止**
「新規行が作成された（トリガー発火）」瞬間と「外部APIからセルの値（URL）が実際に書き込まれる」瞬間には、コンマ数秒のタイムラグが発生することがあります。これを考慮し、プログラムの最初に2秒待機を入れてからデータを読み込むことで、URLの取得漏れ（空欄読み込み）を防ぎます。
3. **二重処理防止（ステータス列の更新）**
スプレッドシートが更新されるたびにプログラムが走るため、すでにGoogle Driveに保存済みの画像に対して、再度ダウンロード処理が走って容量を圧迫するのを防ぎます。一度処理した行には `保存済` というマークをつけ、それを検知してスキップするようにしています。

この設定を行うことで、外部APIによりURLを記録されたから送信された画像が自動的かつ確実にGoogle Driveへバックアップされるようになります。まずはご自身のテストアカウント等でフォームを送信し、動作を確認してみてください。
